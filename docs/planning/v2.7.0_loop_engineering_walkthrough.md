## Deviations from the plan

- `loop` and `curate` subcommands live in `cli/bin/ace-framework.js` (the
  in-project CLI), not `create-ace-framework.js` (the scaffolder) as the
  plan's T004 file list implied — the scaffolder gained `--adapter` instead.
- `loop --dry-run` with no queue exits 0 with guidance (the plan's smoke
  test assumed a preview should not error).
- Found and fixed during the dogfood drill: the manual runner now returns a
  runner error when stdin closes before confirmation, instead of letting
  node exit silently mid-loop (state remained recoverable via resume either
  way).
- `.gitattributes` gained `*.sh eol=lf` so the enforced hooks survive
  Windows checkouts.

## Known limitations (v2.8 candidates)

- Single-task-at-a-time only (parallel Generators deferred per plan Open
  Items — reopens the lock-awareness problem).
- The `claude-code` runner has unit coverage via injected spawn but no live
  headless session in this repo's CI.
- Scaffold-time `--adapter` verification end-to-end requires the pushed
  repo (the scaffolder clones from GitHub main); re-run after push:
  `npx create-ace-framework tmp --adapter claude-code`.
- Reflector/Curator automation is marked **Experimental** in ACE-SPEC §13.4.
